# /king-onboard

Guía al usuario por el ciclo completo del SDLC de King Framework usando su codebase real. Cubre las 9 fases principales: genesis, brainstorm, plan, build, review, qa y merge — con narración educativa en cada paso.

## Uso

```
/king-onboard
```

## Qué hace

1. Escanea el estado actual del proyecto
2. Bootstrapea con /genesis si el proyecto no está inicializado
3. Guía por un cambio pequeño y real a través del ciclo completo
4. Narra qué hace cada skill en cada fase y por qué
5. Menciona skills utilitarios en contexto (refine, worktree, test-plan, etc.)
6. Presenta el mapa completo de los 42 skills al finalizar

## Cuándo usarlo

- Cuando alguien nuevo al equipo necesita entender el SDLC de King Framework
- Cuando querés explorar el ciclo completo en un proyecto nuevo
- Como punto de entrada antes de usar los skills individualmente

## Notas

- El onboarding trabaja con un cambio REAL de 30-60 minutos — no ejercicios ficticios
- Pausa en Fase 3 (brainstorm) y Fase 4 (plan) para revisión antes de implementar
- Para aprender específicamente SDD: `/sdd-onboard`
- Mapa completo de skills disponible en: `skills/king-onboard/REFERENCE.md`

Usa: `skills/king-onboard/SKILL.md` + `skills/king-onboard/PHASES.md`
